## Confirmed Findings And Fixes

### `surface-113`: Allowlisted localhost `--url` targets were blocked as unsafe

Repro:

- Temp `.surface/config.yml` allowed `http://127.0.0.1:59172`.
- `surface capture --url http://127.0.0.1:59172/settings/profile --json` exited 1 with `target_not_allowed` and `reason: unsafe-host`.
- `surface capture --url http://localhost:59172/settings/profile --json` and `surface audit --url http://127.0.0.1:59172/checkout --json` failed similarly.

Fix:

- `authorizeCaptureTarget` now evaluates allowlist matching before unsafe-host rejection.
- Only explicitly allowlisted canonical local dev hosts (`localhost`, `127.0.0.1`, `::1`) can bypass the URL unsafe-host block.
- Alternate loopback and private URL hosts remain blocked.

Verification:

- Added acceptance coverage for allowlisted `127.0.0.1` and `localhost` URL targets.
- Existing URL safety tests still pass.
- Live after-fix commands exited 0 for capture and audit.

### `surface-dcy`: QA reports were awkward for automation and humans

Repro:

- `surface report qa --run qa_checkout_2b04c47b --format json --json` returned `data.report` as a serialized JSON string.
- `surface report qa --run qa_checkout_2b04c47b --format md` printed `surface report qa: {"format":"md","report":"..."}` instead of Markdown.

Fix:

- `reportQa(format: json)` now returns the structured report manifest object.
- The CLI writes Markdown report bodies directly for non-JSON Markdown output.
- JSON mode still uses the standard command envelope.

Verification:

- Added focused core and CLI tests.
- Live after-fix `--format json --json` returned an object at `data.report`.
- Live after-fix `--format md` printed a Markdown document beginning `# Browser QA Report ...`.

### `surface-0a1`: Local link docs failed when `surface` already existed

Repro:

- README Local Development instructed `pnpm --filter @zigrivers/surface exec npm link`.
- In this environment that failed with `EEXIST` because `/opt/homebrew/bin/surface` already existed.

Fix:

- README now advises avoiding accidental overwrite of package-manager-owned binaries and gives
  `node packages/cli/dist/index.js --help` as a safe one-off local smoke test.

## Remaining Risk

`surface-437` tracks the remaining backlog UX ambiguity. After a default audit recorded one content
finding, scoped zero-finding audits for visual hierarchy and responsiveness preserved that finding,
but the top-level backlog was rewritten as `backlog_run_mq7yzpei_s8uyro` with `runId:
run_mq7yzpei_s8uyro`, even though the listed finding originated in `run_mq7yyaf9_9trpr1`. This may
be consistent with persistent active-backlog semantics, but the JSON is ambiguous for automation.

Also noted: local deterministic lenses such as content and visual hierarchy can emit findings with
`method: "judged"` even when model egress is blocked. This appears intentional in the current lens
registry, but it makes the README phrase "measured-only by default" easy to misread.
